# XMUOJ VS Code 插件

这是 XMUOJ VS Code 插件的当前实现版本，目标是让学生在 VS Code 里完成找题、读题、本地调试和提交。当前这轮重构参考了 LeetCode 官方插件的交互思路，把重点放在“分区工作台 + 编辑器内快捷操作 + side mode 布局 + 状态徽标 + 提交历史 + 工作台恢复”。

## 已实现功能

- 快速开始入口，适合第一次使用时按步骤完成初始化
- 通过 `/api/plugin/login` 使用插件令牌登录
- 浏览比赛列表
- 浏览公开题库
- 通过比赛 ID 或比赛链接打开比赛
- 在侧边栏以“快捷入口 / 当前作答 / 最近记录 / 比赛题目”的分区工作台组织内容
- 在题面页和侧边栏提供“开始作答”入口，自动恢复或创建本地代码文件
- 重新打开最近访问的比赛
- 重新打开最近访问的题目
- 在编辑器顶部提供类似 LeetCode 的 CodeLens 快捷操作，例如查看题面、运行测试、提交评测；管理员还会看到 AI解题、AI代码
- 启用 side mode 时，题面固定在主列，代码与结果面板优先在旁边列打开
- 在题目树中通过图标、颜色和状态徽标强化显示，例如 TODO、DOING、LOCAL、SUBMIT、AC
- 使用网页面板查看题面、样例和测试数据清单
- 在本地创建题目工作区，并生成 `.xmuoj.json` 元数据
- 下载并解压测试数据到本地工作区
- 按比赛批量生成本地题目目录，适合课堂实验或机房发题
- 对 C、C++、Java、Python3 运行本地样例或测试数据
- 在题目列表中显示作答状态，例如未开始、本地通过、已提交、已 AC
- 显示本地验收报告，逐个测试点展示期望输出和实际输出
- 本地测试通过后可直接从提示里“立即提交”，减少切换步骤
- 生成本地任务，便于一键编译和运行样例
- 在题目工作区里切换语言模板
- 题面操作栏使用紧凑图标按钮，减少横向占位，悬停可查看操作名称
- 管理员在题面页可使用“AI解题”“AI代码”按钮，分别生成中文题解和对应语言代码
- 提交当前文件并轮询判题结果
- 使用统一结果面板查看最近一次本地测试、最近一次在线提交和最近提交历史
- 在“最近记录”分区里快速回看最近几次提交结果
- 重新打开 VS Code 后自动恢复最近一次题面、代码文件和结果面板布局

## 配置项

- `xmuoj.baseUrl`：XMUOJ 站点地址，例如 `http://www.xmuoj.com`
- `xmuoj.allowInsecureTls`：是否允许连接自签名证书站点，仅建议在校内测试环境使用
- `xmuoj.editor.shortcuts`：编辑器顶部显示哪些快捷操作，默认会显示开始作答、运行测试、提交评测、查看题面、结果面板；管理员还会自动显示 AI解题、AI代码
- `xmuoj.enableSideMode`：是否启用 side mode 列布局
- `xmuoj.colorizeProblems`：是否在题目树中启用状态图标和颜色强化

## 使用说明

1. 安装插件后，先执行 `XMUOJ：快速开始` 或 `XMUOJ：设置站点地址`。
2. 如果你的站点不是默认的 `http://www.xmuoj.com`，先把 `xmuoj.baseUrl` 改成学校实际使用的地址。
3. 执行 `XMUOJ：登录`，输入账号密码。
4. 通过 `XMUOJ：浏览比赛` 或 `XMUOJ：浏览题库` 找到题目。
5. 打开题目后，优先执行 `XMUOJ：开始作答`，插件会自动恢复或创建本地代码文件。
6. 如果老师开放了测试数据，可以执行 `XMUOJ：下载测试数据`。
7. 写完代码后，执行 `XMUOJ：运行本地测试`。
8. 本地通过后，可以直接点击提示里的“立即提交”，也可以手动执行 `XMUOJ：提交当前文件`。

## 补充说明

- 打开比赛时，如果比赛有密码，可以直接输入。
- 提交时会优先根据当前文件后缀判断语言，不行时再回退到 `.xmuoj.json` 中保存的语言。
- 本地测试依赖你当前机器上的编译器或解释器环境。
- 如果插件提示证书错误，而你的学校站点使用的是自签名证书，可以开启 `xmuoj.allowInsecureTls` 后重试。
- 如果插件提示 `/api/plugin/` 返回 404，说明当前站点还没有部署插件后端接口，单纯改关键词或重新登录都不会生效。
- `AI解题` 和 `AI代码` 当前仅对管理员展示，且依赖本机可用的 GitHub Copilot Chat 语言模型能力。
- 最近比赛、最近题目、最近一次提交结果、最近一次本地报告、最近提交历史和工作台布局都会保存在 VS Code 全局状态中。
- 批量生成比赛工作区会为每道可见题目创建本地目录，并写入对应任务。
- 插件依赖后端新增的 `/api/plugin/` 接口。
